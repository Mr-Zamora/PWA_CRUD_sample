How to Recreate This Website

This guide provides step-by-step instructions for recreating this Filipino Recipes website, focusing on modern web development best practices.

Core Web Technologies

  • HTML5 Semantic Structure: Use semantic tags like <header>, <nav>, <main>, <section>, <article>, and <footer> to create a meaningful document outline that improves accessibility and SEO.
  • CSS Layout Systems: Implement Flexbox for one-dimensional layouts (header, navigation, footer) and CSS Grid for two-dimensional layouts (recipe card grid).
  • Responsive Design: Use media queries and relative units (rem, %, vh/vw) to ensure the site works on all device sizes.
  • Progressive Enhancement: Build the core functionality first, then add advanced features for modern browsers.

Pages and Component Structure

1. Index Page (index.html)

Purpose: Display a responsive grid of recipe cards with clickable links to detailed recipe pages.

  • Header Component:
    • Structure: <header class="header" role="banner"> with <div class="logo"> and <nav class="nav-links" aria-label="Main navigation">
    • Layout: Use Flexbox with display: flex; justify-content: space-between; to position logo left and navigation right
    • Mobile Toggle: <div class="nav-toggle" aria-label="Toggle navigation menu" role="button" tabindex="0"> for hamburger menu
  • Recipe Grid Component:
    • Structure: <main class="content grid-4-col" role="main"> containing multiple <article class="card"> elements
    • Layout: CSS Grid with display: grid; grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)); gap: 20px;
    • Card Structure: <header class="card-header"> with category and title, followed by <div class="card-body"> with description
    • Data Attributes: data-category="Main Course" for potential filtering functionality

2. Recipe Detail Page (recipe-detail.html)

Purpose: Display detailed information about a specific recipe with a responsive two-column layout.

  • Header Component: Reused from index.html (component consistency)
  • Recipe Detail Component:
    • Structure: <main class="recipe-content" role="main"> with two columns
    • Layout: CSS Flexbox with display: flex; gap: 2rem; for desktop and flex-direction: column; for mobile
    • Image Column: <img class="recipe-image" src="images/recipe.jpg" alt="Descriptive text"> with max-width: 100%; height: auto; for responsiveness
    • Content Column: <div class="recipe-details-container"> containing:
      • Title: <h1>Recipe Title</h1>
      • Description: <p class="description">
      • Ingredients: <h2>Ingredients</h2><ul class="ingredients-list">
      • Directions: <h2>Directions</h2><ol class="directions-list">
      • Action Buttons: <div class="action-buttons"><a href="#" class="action-link edit">Edit</a><a href="#" class="action-link delete">Delete</a></div>

3. Add Recipe Page (add-recipe.html)

Purpose: An accessible, validated form for users to submit new recipes with proper feedback.

  • Header Component: Reused from index.html
  • Form Component:
    • Structure: <main class="form-container" role="main"><form id="recipe-form" class="recipe-form" novalidate>
    • Accessibility: Each input has associated <label for="input-id"> and appropriate ARIA attributes
    • Form Fields:
      • Category: <select id="recipe-category" name="category" required aria-required="true"> with options
      • Recipe Name: <input type="text" id="recipe-name" name="recipe-name" required aria-required="true" placeholder="Enter recipe name">
      • Description: <textarea id="recipe-desc" name="recipe-desc" rows="3" required aria-required="true" placeholder="Brief description of the recipe"></textarea>
      • Ingredients: <textarea id="ingredients" name="ingredients" rows="5" required aria-required="true" placeholder="Enter ingredients, one per line"></textarea>
      • Directions: <textarea id="directions" name="directions" rows="5" required aria-required="true" placeholder="Enter step-by-step directions"></textarea>
      • Image Upload: <input type="file" id="image-upload" name="image-upload" accept="image/*" aria-describedby="file-format-help"> with help text
    • Form Validation: Client-side validation with :invalid CSS and JavaScript validation on submit
    • Buttons: <div class="form-buttons"><button type="submit" class="btn primary">Add Recipe</button><button type="button" class="btn secondary" onclick="window.location.href='index.html'">Cancel</button></div>

4. About Page (about-this.html)

Purpose: Provide a description of the project, its technical specifications, and educational value.

  • Header Component: Reused from index.html
  • About Content Component:
    • Structure: <main class="about-content" role="main"> with two-column layout on desktop
    • Image Section: <img class="about-image" src="images/about.webp" alt="Filipino cuisine ingredients">
    • Text Section: <div class="about-text"> containing:
      • Title: <h1>About the Filipino Recipes Project</h1>
      • Project Description: <p class="project-intro"> explaining the purpose and goals
      • Tech Stack: <h2>Technology Stack</h2><ul class="tech-list"> listing HTML5, CSS3, JavaScript
      • Design Principles: <h2>Design Principles</h2><ul class="principles-list"> explaining responsive design, accessibility, etc.
      • Future Development: <h2>Future Development</h2><p> explaining the transition to a Flask-based PWA

5. CSS Organization (styles.css)

Purpose: Create a maintainable, responsive design system with consistent styling across all pages.

CSS Architecture

The CSS is organized into logical sections with clear comments to make it easy to find and modify styles:

  • CSS Variables (Custom Properties):
    • :root { --primary-color: rgb(174, 60, 60); --text-color: #333; --light-bg: #f9f9f9; }
    • Benefits: Consistent colors, easy theme changes, better maintainability
  • Base Styles:
    • * { box-sizing: border-box; } for predictable sizing
    • body { font-family: Arial, sans-serif; line-height: 1.6; margin: 0; }
    • h1, h2, h3 { font-family: 'Asar', serif; } for typographic hierarchy
  • Layout Components:
    • .header and .footer using display: flex; justify-content: space-between; align-items: center;
    • .grid-4-col using display: grid; grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)); gap: 20px;
    • .recipe-content using display: flex; gap: 2rem; for two-column layout
  • Component Styles:
    • Card components: .card, .card-header, .card-body
    • Form components: .form-group, .btn, .btn.primary, .btn.secondary
    • Navigation: .nav-links, .nav-toggle, .logo
  • Responsive Design:
    • Mobile-first approach with base styles for small screens
    • Media queries: @media (min-width: 768px) { /* Desktop styles */ }
    • Responsive typography using rem units
    • Fluid layouts with percentage widths and max-width constraints
  • Accessibility Features:
    • Focus styles: :focus { outline: 3px solid var(--primary-color); }
    • Sufficient color contrast (WCAG AA compliance)
    • Skip-to-content link for keyboard navigation

6. JavaScript Functionality

Purpose: Enhance the user experience with interactive features while maintaining accessibility.

  • Mobile Navigation Toggle:
    • Select elements: const toggleButton = document.querySelector('.nav-toggle');
    • Event listener: toggleButton.addEventListener('click', () => { navLinks.classList.toggle('show'); });
    • Accessibility: Update ARIA attributes with toggleButton.setAttribute('aria-expanded', isExpanded);
  • Form Validation:
    • Prevent default submission: form.addEventListener('submit', (e) => { if (!form.checkValidity()) { e.preventDefault(); } });
    • Custom validation messages: input.setCustomValidity('Please enter a valid recipe name');
    • Visual feedback: Add/remove .error class based on validation state
  • Future Enhancements:
    • Recipe filtering by category
    • Local storage for saving form drafts
    • Dynamic recipe card generation from JSON data

7. Transitioning to Flask

When ready to convert this static site to a Flask web application:

  • Project Structure: Move HTML files to templates/ folder and static assets to static/ folder
  • Template Conversion: Convert HTML files to Jinja2 templates with {% extends "base.html" %}
  • URL Generation: Replace static links with {{ url_for('route_name') }}
  • Static Assets: Update paths with {{ url_for('static', filename='css/styles.css') }}
  • Data Management: Replace hardcoded content with template variables {{ recipe.name }}
